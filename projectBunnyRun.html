When: Feb. 2020 - May 2020 (3 months)

Class: Fundamentals of Programming

Team: Me, Myself, and I

Tech: Python (pygame)

Focus: Competitive Analysis, Game Development, Game GUI Design, Storyboarding

Purpose

As someone who grew up on Super Mario Bros., platformers games are certainly an important part of my childhood. Therefore, when I was told that I can create anything for my final project for the Fundamentals of Programming class, I knew immediately that I want to design and create my own platformer game. My goal for the project was simple: creating a user-friendly game with an aesthetically pleasing GUI that brings people joy.

Competitive Analysis

Platform games are a common genre of video games. At the peak of their popularity, it is estimated that between one-quarter and one-third of console games were platformers, but the genre has become far less dominant in the market as action and shooter games rise in popularity (citation). Nevertheless, classic platformer games such as Super Mario Bros. and Fireboy & Watergirl remain well-known among people. Some features that make Super Mario Bros. unique is its classic sound effects and background music, various types of enemies, and the use of pipe as a transportation method to other places. On the other hand, although Fireboy & Watergirl has simpler graphic designs compared to Super Mario Bros., the biggest advantage the game has is that it allows for multiplayer.

Storyboard

Before I start to write any code, I wanted to brainstorm and create a storyboard to plan and visualize every stage of my game. while designing the game, I tried to avoid discontinuity in the player’s experience, be consistent with my design choice, and breathe life into my user interface using visual and audio cues.

The features I wished to implement were:

  1. User log-in
  2. Shop
  3. Scoreboard
  4. Coin
  5. Mystery box
  6. Maze
  7. Various enemies
  8. Bullet
  9. Sound effect

Implementation & Walkthrough

I used Python and pygame library to develop the game. Specifically, I used the concept of object-oriented programming to create different game screens, enemy behaviors, and player movements. You can view the code here.

Reflection

This was one of the most challenging, but rewarding projects I had worked on up to this point. Since I didn’t know about the pygame library beforehand, I had to do a lot of research to familiarize myself with the library. Moreover, I learned a lot from the process of translating my vision into code to create a playable game, scripting functions, events, and interactions, identifying and fixing bugs, and creating the User Interface. From this project, I learned to consider game user experience and be mindful that I am creating a project not for myself but for other players (think about the final users for the game). I learned to avoid discontinuity in the player’s experience, be consistent with my design choice, and breathe life into my user interface using visual and audio cues.
